Entenda de vez como cada valor funciona — com demos visuais e código.

static — padrão do navegador

Todo elemento começa com position: static. O elemento segue o fluxo normal do documento. As propriedades top, right, bottom e left não têm efeito.

/* Padrão — não precisa nem declarar */ .caixa { position: static; /* top / left não funcionam aqui */ }
Use quando não precisa alterar o posicionamento.

relative — relativo a si mesmo

O elemento se move em relação a onde ele estaria normalmente. O espaço original é preservado no layout (o fantasma abaixo mostra isso).

.caixa-2 { position: relative; top: 20px; /* desce 20px do lugar original */ left: 30px; /* vai 30px para direita */ }
Muito usado como âncora para filhos com position: absolute.

absolute — relativo ao ancestral posicionado

O elemento sai do fluxo (não ocupa espaço). Ele se posiciona em relação ao ancestral mais próximo que tenha position diferente de static. Se não existir nenhum, usa o <html>.

Sem um pai posicionado, o filho absolute escapa para o canto do <html> — cuidado!

fixed — relativo à janela do navegador

O elemento sai do fluxo e fica preso à viewport (janela). Não se move com a rolagem da página. A navbar no topo desta página é um exemplo real.

janela do navegador (simulada)
+ Novo
.botao-flutuante { position: fixed; bottom: 24px; right: 24px; z-index: 999; }
Ideal para navbars, chats flutuantes, botões de ação principal (FAB).

sticky — híbrido: relative + fixed

O elemento age como relative até atingir o limite definido (ex: top: 0), depois "gruda" como fixed — mas apenas dentro do seu contêiner pai. Role a lista abaixo para ver o cabeçalho grudando.

Grupo A — gruda no topo ao rolar
Item A1 — Lorem ipsum dolor sit
Item A2 — Consectetur adipiscing
Item A3 — Sed do eiusmod tempor
Item A4 — Ut labore et dolore
Grupo B — também gruda
Item B1 — Quis nostrud exercit
Item B2 — Ullamco laboris nisi
Item B3 — Duis aute irure dolor
Item B4 — Excepteur sint occaecat
Grupo C
Item C1 — Sunt in culpa qui
Item C2 — Officia deserunt mollit
.cabecalho-grupo { position: sticky; top: 0; /* gruda quando chega ao topo */ }
O pai precisa ter altura e overflow adequados. Se o pai for pequeno demais, o efeito não aparece.

Resumo comparativo

Valor Sai do fluxo? Referência top/left funcionam? Rola com a página?
static Não Fluxo normal Não Sim
relative Não (reserva espaço) Posição original Sim Sim
absolute Sim Ancestral posicionado Sim Sim (com o pai)
fixed Sim Viewport (janela) Sim Não — fica fixo
sticky Não → Sim (no limiar) Contêiner pai Sim (define o limiar) Até o limiar, depois gruda
